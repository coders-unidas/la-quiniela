1. Introduction

The aim of this project is to develop a machine learning model simulating a Quiniela game using historical data from LaLiga. The model can give you the predictions of the results of the matches for a given match day in a certain division an season.

The main objective is to give the model features so they can predict an outcome, called target. The model is trained with data from those features and in which target did they result. Once the model is trained, if new features are introduced, it will predict a certain target.

In the case of this project, the target will be whether the home team wins, loses or if there is a tie. The outcome of the program will be: 1 if the home team wins, 2 if the away team wins or x if there is a tie. In order to see the performance of the model, matches which the score is already already known can be predicted, so the prediction can be compared with the reality and establish the fiability of the model.

2. Description of the model

2.1 Features

We have established three candidate features for our model that are features related to the outcome of the matches. The first feature is which team is home team and which one is away team. During the Data Analysis of LaLiga we observed that the team that is playing at home has more tendency to win rather than team that is playing away, so we decided that it would be an interesting feature to take into consideration. In Figure 1 it can be seen the huge tendency there is.

Secondly, another thing that is quite related to which team has more odds of winning a match is how high or low are they in the rank position; the higher a team is positioned in the rank means that in the past they performed really well. So, thinking that a team with a higher rank playing against a team with a lower rank will win, is quite a safe bet. This is the login between why whe chose the following features: the rank position from the latter position and the rank position in the current season each day (in other words, how are they positioned in the rank a certain day of the season according to the matches until that day). To take into consideration the division shifts, we established that the rank position in division 2 is N+n (the last rank position in division 1 of the season plus the rank position of the team in division 2).

In order to choose which features we chose to develop the model, we did three Jupyter Notebooks that can be found in the folder analysis.
